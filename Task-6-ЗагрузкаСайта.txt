Ввод в адресную строку reg.ru
Браузер начинает искать IP сервера для загрузки сайта. 
Данная информация хратиться в системе DNS (Domain Name System). Браузер пытается найти запись об IP сайта в ближайших местах для экономии времени:
1. Сначала в своей истории подключений (кэш браузера). 
2. В кэше роутера, который сохраняет информацию о последних соединениях, совершенных из локальной сети.
3. Далее браузер формирует запрос к DNS-серверам начиная с корневой зоны ("невидимая"  точка справа), далее проверяет 1, 2, 3... уровни домена, пока не получит IP для нужного домена.
4. Узнав IP пытается установить с ним соединение используя специальный протокол — TCP (набор правил описывающих способы соединения между устройствами, форматы отправки запросов, действия в случае потери данных и т.д.)
5. Браузер отправляет HTTP-запрос, чтобы получить контент сайта
В этом запросе содержится информация о самом браузере, временные файлы, требования к соединению и т.д.
В общении браузера и сервера выделяют два типа запросов. GET-запрос используется для получения данных с сервера (отобразить картинку, текст или видео). POST-запрос — используется для отправки данных из браузера на сервер (отправляет сообщение, загружает файл).
Обмениваются информацией с сервером производить с помощью HTTPS -зашифроывнный протокол с использованием ключей SSL установленных на сервер. В нашем случае это сертификат от GlobalSign, типа EV. 
6. Сервер получил запрос от браузера и обрабатывает его по средствам веб-сервера nginx. После этого запрос перенаправляется на дальнейшую обработку к программе-обработчику ( PHP). По запросу определяет содержимое (в каком формате и какие именно файлы нужно отпрвить). Далее формируется ответ.
7. После формирования ответа он отправляется веб-сервером обратно браузеру. В ответе содержится контент для отображения веб-страницы, информация о типе сжатия данных, способах кэширования, файлы cookie, которые нужно записать и т.д.
8. После получения ответа начинается процесс рендеринга, т.е. распаковывает  полученный ответ и отображает полученный контент на экране.
Сначала загружает только основную структуру HTML-страницы. Затем последовательно проверяет все теги и отправляет дополнительные GET-запросы для получения с сервера различных элементов — картинки, файлы, скрипты, таблицы стилей. Параллельно с этим на компьютер сохраняются статичные файлы пользователя, т.е кэшируются. 
После завершения рендеринга отображается страница целиком со всеми элементами.
